问题1 - 特征观察¶

如前文所述，本项目中我们关注的是其中三个值:'RM'、'LSTAT' 和'PTRATIO'，对每一个数据点:

  • 'RM' 是该地区中每个房屋的平均房间数量；
  • 'LSTAT' 是指该地区有多少百分比的房东属于是低收入阶层（有工作但收入微薄）；
  • 'PTRATIO' 是该地区的中学和小学里，学生和老师的数目比（学生/老师）。

凭直觉，上述三个特征中对每一个来说，你认为增大该特征的数值，'MEDV'的值会是增大还是减小呢？每一个答案都需要你给出理由。

回答:

  1. 增大RM,MEDV增大,即房屋数量增加,价格也会增加
  2. 增大LSTAT,MEDV减小,多数房东收入低,房价自然低
  3. 增大PTRATIO,MEDV增大,因为学生人数增加,老师人数相对减小,教师买房子有教育优惠,学生没有,所以房价会增加

问题 8- 交叉验证¶

什么是K折交叉验证法（k-fold cross-validation）？优化模型时，使用这种方法对网格搜索有什么好处？

提示： 跟为何需要一组训练集的原因差不多，网格搜索时如果不使用交叉验证会有什么问题？

答案: k-fold cross-validation是把训练数据集随机等分成k组,之后运行k次训练与测试,选择k-1组用于训练,选择1组用于测试模型预测分数,每次的测试集选择均不相同,然后对k次测试结果取平均值作为测试结果. 使用K折交叉验证法,可以从某种意义保证上全部数据集都作为训练集和测试集,大大提高了模型评估的准确率.这样的话,网格搜索便能很准确低找出模型的最佳参数.如果简单得把数据分成训练和测试组的话,训练集和测试集太局限,最后的模型泛化能力可能不足,而且模型评估不准确,以为受到数据集划分的随机性的影响.

练习：训练模型¶

在最后一个练习中，你将需要将所学到的内容整合，使用决策树演算法训练一个模型。为了保证你得出的是一个最优模型，你需要使用网格搜索法训练模型，以找到最佳的 'max_depth' 参数。你可以把'max_depth' 参数理解为决策树算法在做出预测前，允许其对数据提出问题的数量。决策树是监督学习算法中的一种。

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': range(1,11)}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search object
    grid = GridSearchCV(regressor,params,scoring=scoring_fnc,cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

问题 11 - 实用性探讨¶

简单地讨论一下你建构的模型能否在现实世界中使用？

提示： 回答几个问题：

  • 1978年所采集的数据，在今天是否仍然适用？
  • 数据中呈现的特征是否足够描述一个房屋？
  • 模型是否足够健壮来保证预测的一致性？
  • 在波士顿这样的大都市采集的数据，能否应用在其它乡镇地区？

答案: 构建的模型不能在现实世界使用.

  1. 不适用,因为物价和人们的消费水平,以及其他外来移民,附近建筑或者自然灾害等地理位置多种影响因素会发生改变.
  2. 不够.还有周围环境,交通便捷,等因素的影响.
  3. 是.这十次结果的平均值为\$404,978.717,标准差为\$19,894.17,可见模型预测的一致性足够好.
  4. 不能应用在其他乡镇地区.受多种因素影响,它们房价水平不一样. 总结来说,数据不足够代表性,特征较少而使模型不足以在现实世界中广泛使用.
